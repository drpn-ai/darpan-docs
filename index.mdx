Darpan is a reconciliation workspace. It connects source systems, describes source data with schemas, executes reconciliation runs, applies RuleSets, and preserves the output needed to explain each difference.

Most of the work in Darpan is front-loaded: connections, schemas, and a saved run are set up once and reused. The docs follow that shape — setup records first, then execution, then output and follow-up. If this is your first time, the quickstart below walks the full path end to end.
