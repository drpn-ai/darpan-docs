---
title: "Configure AI providers"
description: "Set up AI provider records used by assisted Darpan workflows."
---

AI provider setup is a Super Admin task you do once per provider. Save the record with the right credential reference and Darpan's assisted workflows — schema support, analysis, and similar features — have what they need from then on.

Tenant Admins manage tenant-owned setup records such as timezone, notifications, connections, schemas, saved runs, and automations. AI provider settings are outside their scope.

## Before you start

Confirm:

- You are signed in as a Super Admin.
- The provider identifier and required credential reference are available in secure configuration.

## Setup checklist

1. From **Ask Darpan**, open **Open AI Settings**.
2. Create or edit the provider record.
3. Enter the provider identifier and required credential reference.
4. Save the record.
5. Run the smallest workflow that uses the provider.

## Record naming

Clear names and descriptions show whether a provider is for local testing, pilot use, or a production-like target.

## Safety

Credentials belong in the configured provider record. Public examples use credential references, not raw keys, organization identifiers, or model-routing details.
