title: "Connections"
description: "Troubleshoot external connection records."
---

A connection that worked last week can break after a credential rotation, a host change, or a server that stopped accepting the authentication method it was set up with. The checks below are grouped by connection type. If you haven't set up the connection yet, start with the setup guide for that type instead.

## SFTP

Setup guide: [Connect SFTP](/guides/connect-sftp).

Search **Ask Darpan** for **Open SFTP Servers**. For an existing record, search by SFTP description, host, or username. Check host, port, username, authentication method, directory path, and network access. Confirm whether the server expects a private key, password, or another authentication method.

## NetSuite

Setup guide: [Connect NetSuite](/guides/connect-netsuite).

Search **Ask Darpan** for **Open NetSuite Settings**. Check account target, role permissions, token validity, integration status, and RESTlet script deployment.

## AI providers

Setup guide: [Configure AI](/guides/configure-ai).

Search **Ask Darpan** for **Open Tenant Settings** (AI provider settings live on the Tenant Settings page). Check the provider record, credential reference, model availability, and feature-specific limits.

## Safe diagnostics

Capture error codes and messages with secrets and organization-specific identifiers redacted before sharing externally.
