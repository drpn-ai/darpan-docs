A reconciliation run is one execution of a reconciliation.

## Run data to preserve

- Who started the run
- Active company or active tenant
- Selected schemas for source shape
- RuleSet or rules used for comparison logic
- Source inputs
- Processing status
- Generated output references
- Error details when processing fails

## Status interpretation

| Status | Meaning |
| --- | --- |
| Pending | The run record exists but processing has not started or completed. |
| Running | Darpan is processing inputs. |
| Succeeded | Processing finished and output is ready to review. |
| No data | Processing finished but no input data was available. |
| Failed | Processing stopped before producing expected output. |
| Skipped duplicate | The run matched an existing run and was skipped as a duplicate. |

## Review principle

Treat the run as evidence. If you change schema, mapping, source files, or rules, create a new run instead of overwriting the old result.
